# PTA 产业分析系统 - 项目方案

> 版本：v0.1
> 日期：2026-04-10

---

## 一、项目愿景

构建一个以 **PTA（精对苯二甲酸）为核心的产业链数据分析与洞察系统**。

核心逻辑：

```
期货价格 ←→ 现货价格 ←→ 基差 ←→ 仓单
         ↑                              ↓
         ←←←←← 供给 / 需求 ←←←←←←←←←←
                       ↓
              上市公司公告（文本）
                       ↓
              AI 分析 → 产能信号提取
                       ↓
              供需格局判断 → 价格方向结论
```

两层分析：

1. **数据层**：价格数据 → 图表可视化 → Agent 分析 → 结论
2. **文本层**：公告文本 → 单篇解析 → 时序综合 → 趋势判断

## 四、功能模块设计

### 4.1 模块一：数据仪表盘

**目的**：展示 PTA 期货品种的核心数据变化，回答"现在市场是什么情况"。

#### 4.1.1 数据绑定

针对目标品种（如 TA409）：

```
期货价格：fut_daily  →  收盘价、成交量、持仓量
现货价格：price_data →  对应品种的现货价
基差：basis_factor_daily → 现货价 - 期货价
仓单：fut_wsr → 注册仓单量、变化量
```

#### 4.1.2 图表类型

| 图表 | 数据 | 用途 |
|------|------|------|
| K 线图 | 期货日线 | 看价格趋势 |
| 基差走势图 | 现货 - 期货 | 基差收敛/扩大，预警供需拐点 |
| 仓单量柱状图 | fut_wsr | 仓单增加=现货压力，仓单减少=现货紧张 |
| 现货-期货-基差叠加图 | 三者叠加 | 快速判断价差结构 |
| 价格与仓单相关性图 | 价格 vs 仓单量 | 辅助判断方向 |

#### 4.1.3 Agent 分析逻辑（提示词）

输入：上述图表数据（数值序列）

输出：结构化结论

```
提示词模板（data_agent_prompt）：

你是一个PTA期货品种的数据分析师。以下是{品种}最近20个交易日的核心数据：

期货数据：
- 日期、收盘价、成交量、持仓量

现货数据：
- 日期、现货价格

基差数据：
- 日期、基差值

仓单数据：
- 日期、仓单总量、仓单变化

请分析：
1. 【基差研判】当前基差处于历史什么分位？基差扩大还是收窄？说明了什么？
2. 【仓单信号】仓单量近期变化趋势？增加还是减少？结合基差看，说明了现货的什么问题？
3. 【量仓分析】成交量和持仓量的变化？量价配合是否健康？
4. 【综合结论】综合以上信息，当前后市价格方向判断（上涨/下跌/震荡），理由3点。

输出JSON格式：
{
  "basis_signal": "基差信号描述",
  "warehouse_signal": "仓单信号描述", 
  "volume_signal": "量仓信号描述",
  "direction": "看涨/看跌/震荡",
  "confidence": "高/中/低",
  "reasoning": ["理由1", "理由2", "理由3"]
}
```

### 4.2 模块二：公告分析系统

**目的**：从海量公告中提取影响供需的关键信号，判断对 PTA 价格的影响方向。

#### 4.2.1 产业链公司分类

```
PTA 产业链公司分类（供分析用）

【上游 - 炼化/PX】
  荣盛石化(002493)
  恒逸石化(000703)
  恒力石化(600346)
  东方盛虹(000301)  ← 同时有聚酯

【下游 - 聚酯长丝】
  桐昆股份(601233)
  新凤鸣(603225)

【下游 - 瓶片/切片】
  万凯新材(301216)
  三房巷(600370)
```

#### 4.2.2 单篇公告分析（单 Announcement Agent）

输入：一条公告的 `title` + `full_text` + `notice_date` + `f_node_name`

处理流程：

```
公告文本 → LLM → 产能/供需关键信号提取
```

提示词模板（single_notice_prompt）：

```
你是一个PTA产业链的分析师。请分析以下上市公司公告，提取其中对PTA供需可能产生影响的信息。

公司：{stock_name}（{stock_code}）
公告日期：{notice_date}
公告类型：{f_node_name} / {s_node_name}
标题：{title}

公告正文：
{full_text}

请提取以下信息（如果公告中未涉及该信息，填"未提及"）：

1. 【产能信号】
   - 是否有新建产能？产能规模？预计投产时间？
   - 是否有停产/检修/永久关闭？
   - 是否有技术改造影响产量？

2. 【价格信号】
   - 是否有产品售价调整？
   - 原材料成本变化？

3. 【需求信号】
   - 是否有下游需求变化？
   - 是否有出口/进口政策变化影响需求？

4. 【库存/原料信号】
   - 原料采购变化？
   - 库存变化？

5. 【政策/宏观信号】
   - 是否有环保政策影响供给？
   - 是否有宏观需求影响？

6. 【对PTA的影响方向】
   - 供给：增加 / 减少 / 不变 / 不确定
   - 需求：增加 / 减少 / 不变 / 不确定
   - 影响程度：高 / 中 / 低

输出JSON：
{
  "capacity_signal": {...},
  "price_signal": "...",
  "demand_signal": "...",
  "pta_supply_impact": "增加/减少/不变/不确定",
  "pta_demand_impact": "增加/减少/不变/不确定",
  "impact_confidence": "高/中/低",
  "key_conclusion": "一句话总结此公告对PTA最重要的影响"
}
```

#### 4.2.3 时序综合分析（Time-Series Agent）

输入：某品种（如 PTA）一段时间内的所有公告分析结果

处理流程：

```
多个单篇分析结果 → 按时间排序 → 时序Agent → 趋势判断
```

提示词模板（time_series_prompt）：

```
你是一个PTA期货品种的公告分析师。以下是PTA产业链相关公司最近N个月的公告分析结果，按时间倒序排列：

{分析结果列表}

请综合分析：
1. 【整体趋势】从这些公告来看，PTA的供需格局是在变好还是变差？
2. 【关键节点】找出最重要的3个转折性公告，说明理由
3. 【信号汇总】供给端总体是扩张还是收缩？需求端呢？
4. 【结论】综合所有公告信息，对PTA期货未来1-3个月价格方向给出判断

输出JSON：
{
  "overall_trend": "好转/变差/平稳",
  "trend_confidence": "高/中/低",
  "supply_direction": "扩张/收缩/平稳",
  "demand_direction": "扩张/收缩/平稳",
  "key_events": [
    {"date": "...", "event": "...", "significance": "高/中/低"},
    ...
  ],
  "price_outlook": "看涨/看跌/震荡",
  "outlook_reasoning": ["理由1", "理由2"],
  "risk_factors": ["风险因素1", "风险因素2"]
}
```

### 4.3 模块三：综合研判（Final Agent）

**目的**：结合数据层结论 + 文本层结论，给出最终决策建议。

```
综合Agent输入：
1. 数据Agent结论（基差/仓单/价格方向）
2. 时序Agent结论（公告趋势/供需判断）
3. 当前市场核心驱动因素（手动输入或自动提取）

综合分析输出：
- 当前市场主要矛盾
- 核心驱动因素（短期）
- 结论
- 风险提示
```

## 八、关键设计决策

### 8.1 为什么不直接让 AI 读原始数据？

**结构化提取 > 无结构输入**。单篇公告分析时，先用提示词让 LLM 提取"产能信号/价格信号/需求信号"，输出 JSON 字段。这样：
- 输出可存储、可查询、可比较
- 时序分析有了一致的数据结构
- 避免 LLM 每次都重复读原始长文本

### 8.2 为什么要分三层 Agent？

| 层 | 输入 | 输出 |
|---|---|---|
| 数据Agent | 价格/基差/仓单数值 | 短期方向判断 |
| 单篇公告Agent | 单篇公告全文 | 产能/供需信号（结构化）|
| 时序Agent | 多篇分析结果JSON | 中期趋势判断 |
| 综合Agent | 数据Agent结论 + 时序Agent结论 | 最终决策 |

分层的目的是**让每次 LLM 调用专注一个任务**，减少幻觉，提高输出稳定性。

### 8.3 提示词版本管理

提示词是系统的核心资产。采用以下管理方式：

```
prompts/
├── data_agent_v1.md       # 数据分析提示词
├── single_notice_v1.md   # 单篇公告提示词  
├── time_series_v1.md     # 时序分析提示词
└── final_judgment_v1.md  # 综合研判提示词
```

每次调整提示词后，更新版本号，保留历史版本，便于对比效果。

## 九、工作流程（每日运行）

```
1. 收盘后（17:00）
   ↓
2. sync_tushare.py        → 更新期货数据
   ↓
3. sync_notices.py         → 增量同步今日公告
   ↓
4. 【可选】告警判断
   - 如果有重大公告（新产能、检修）→ 触发立即分析
   ↓
5. 数据分析 Agent
   - 输入：今日价格/基差/仓单
   - 输出：短期方向判断
   ↓
6. 公告分析
   - 批量：最近 N 天新公告 → 单篇分析 → JSON 存储
   - 时序：近 30 天分析结果 → 时序 Agent → 趋势判断
   ↓
7. 综合研判 Agent
   - 输入：数据Agent结论 + 时序Agent结论
   - 输出：最终结论
   ↓
8. 输出报告
   - 每日分析文本 → 存入 analysis_results 表
   - 关键信号 → 推送（可选）
```

## 十、待讨论问题

1. **LLM 选择**：使用 OpenAI GPT-4 还是本地模型（如 Ollama）？
2. **分析频率**：每日收盘后分析，还是按需触发（发现重大公告时）？
3. **目标品种**：是否只做 PTA，还是同时支持多个品种？
4. **结果呈现**：输出形式是纯文本报告，还是需要有 Web 界面？
5. **提示词迭代**：如何评估提示词效果？是否需要建立 ground truth 对比集？
